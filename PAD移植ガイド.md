# PAD 移植ガイド (別のPCで動かすためのチェックリスト)

PADフローやスクリプトを別のPCにコピーしただけでは動かない場合があります。以下の 4 つの項目を確認してください。

## 1. Node.js 環境の準備
スクレイピングを実行する Node.js が移植先にもインストールされている必要があります。
1.  [Node.js 公式](https://nodejs.org/) から LTS 版をインストールしてください。
2.  プロジェクトフォルダでターミナルを開き、必要なライブラリをインストールします：
    ```bash
    npm install
    ```

## 2. 環境変数 (.env) のコピー
セキュリティ上の理由から、`.env` ファイルは通常 Git に含まれません。手動でコピーするか新規作成してください。
1.  プロジェクトのルートフォルダに `.env` ファイルを作成します。
2.  以下の内容を記載してください：
    ```env
    # RenderサーバーのURL (Web経由で保存する場合)
    WEB_APP_URL=https://あなたのアプリ名.onrender.com
    # 直接DBに保存する場合のみ必要
    MONGODB_URI=mongodb+srv://...
    ```

## 3. PAD フロー内の「絶対パス」の修正
PAD の「アプリケーションの実行」や「スクリプトの実行」アクションで指定しているパスが、**古い PC のパス（例: d:\prog\...）のまま** になっている可能性があります。
1.  PAD フローを開き、実行ファイルや引数に含まれるフォルダパスを、新しい PC のフォルダ位置に合わせて全て書き換えてください。
2.  `config.json` がスクリプトと同じフォルダにあることも確認してください。

## 4. MongoDB の IP 許可 (直接接続の場合)
もし移植先 PC から直接 MongoDB に書き込む設定にしている場合、MongoDB Atlas の管理画面で **新しい PC のグローバル IP アドレスを Allow List に追加** する必要があります。
※ `WEB_APP_URL` 経由で保存する場合はこの作業は不要です。
